# Contribuindo para AGROTM Solana

Obrigado por contribuir! Siga estas diretrizes para garantir qualidade e colaboração eficiente.

## Como contribuir
- Fork o repositório e crie uma branch descritiva.
- Commits devem seguir o padrão Conventional Commits (ex: feat:, fix:, docs:).
- Abra um Pull Request detalhado e relacione issues relevantes.
- Aguarde revisão de um CODEOWNER.

## Boas práticas
- Escreva código limpo, tipado e documentado.
- Use ESLint e Prettier antes de commitar.
- Não inclua segredos ou dados sensíveis.
- Atualize a documentação se necessário.

## Revisão
- PRs são revisados por pelo menos um CODEOWNER.
- Feedbacks devem ser respeitosos e construtivos.

## Licença
Ao contribuir, você concorda com a licença MIT do projeto.
